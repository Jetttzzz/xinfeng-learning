## 一、LoRaWAN 协议基础

### 1.1 概述
LoRaWAN（Long Range Wide Area Network）是基于LoRa扩频调制技术的低功耗广域网（LPWAN）协议标准，专为物联网设备的远距离、低功耗、低成本通信设计。

### 1.2 网络架构
```
终端设备(End Device) → 网关(Gateway) → 网络服务器(Network Server) → 应用服务器(Application Server)
                                            ↓
                                      Join Server (v1.1)
```

### 1.3 入网方式

#### OTAA（Over-The-Air Activation）- 推荐 ✅
| 参数 | 长度 | 说明 |
|------|------|------|
| **DevEUI** | 64位(16字符十六进制) | 设备全局唯一标识符，类似MAC地址 |
| **AppEUI/JoinEUI** | 64位(16字符十六进制) | 应用服务器/Join服务器标识，同一组设备可相同 |
| **AppKey** | 128位(32字符十六进制) | AES-128根密钥，用于派生会话密钥 |

**入网流程**：
1. 终端发送 Join Request（含DevEUI、AppEUI、DevNonce）
2. 网络服务器验证身份，用AppKey计算MIC校验
3. 服务器下发 Join Accept（含AppNonce、NetID、DevAddr）
4. 终端用AppKey派生会话密钥（NwkSKey、AppSKey）
5. 入网成功，开始加密通信

**优势**：动态协商会话密钥，安全性高，支持漫游

#### ABP（Activation By Personalization）
| 参数 | 长度 | 说明 |
|------|------|------|
| **DevAddr** | 32位 | 网络内设备地址（非全局唯一） |
| **NwkSKey** | 128位 | 网络会话密钥 |
| **AppSKey** | 128位 | 应用会话密钥 |

**特点**：入网速度快，但安全性较低，不支持漫游

### 1.4 设备类别
| Class | 特点 | 适用场景 |
|-------|------|----------|
| **A** | 上行后开两个接收窗口，最省电 | 电池供电设备（水表）✅ |
| **B** | 固定时隙接收窗口 | 需要定时下行的场景 |
| **C** | 接收窗口常开，实时性最好 | 外接电源设备 |

## 六、常见问题排查

### 6.1 网关掉线
**排查步骤**：
1. 检查网关电源和网线连接
2. 检查网关LED指示灯状态
3. 登录网关后台检查网络状态
4. 检查频段配置是否正确
5. 检查MQTT连接状态

### 6.2 水表数据未上报
**排查步骤**：
1. 检查水表是否入网（OTAA成功）
2. 检查网关是否在线且频段匹配
3. 检查平台是否收到数据（查看MQTT日志）
4. 检查水表上报时间窗口与网关状态
5. 使用磁棒触发水表主动上报

**典型案例**（PAM Jaya项目）：
- 问题：水表 0095690600013b18 数据未上报
- 原因：水表定时上报时间（5:13 UTC+8）网关处于离线状态
- 解决：① 磁棒触发主动上报 ② 等待下次定时上报

### 6.3 频段不匹配
**现象**：
- 网关显示在线，但无数据
- 水表无法入网

**解决**：
1. 确认水表频段配置
2. 确认网关频段配置
3. 确保两者一致（如都为AS923-1）

## 九、信号质量判断（RSSI与SNR）

### 9.1 基本概念

#### RSSI（接收信号强度指示）
- **全称**：Received Signal Strength Indication
- **单位**：dBm
- **特点**：负值，**越接近0信号越强**
- **含义**：表示接收端收到的数据包平均信号强度

**示例**：
- `-50 dBm` 的信号比 `-100 dBm` 的信号强

#### SNR（信噪比）
- **全称**：Signal-to-Noise Ratio
- **单位**：dB
- **特点**：正值表示信号比噪声强，负值表示信号比噪声弱
- **含义**：有用信号功率与噪声功率的比值

**SNR判断**：
| SNR值 | 状态 |
|-------|------|
| **SNR > 0 dB** | 信号强度高于噪声，通信质量较好 ✅ |
| **SNR ≈ 0 dB** | 信号与噪声强度相当，通信处于临界状态 ⚠️ |
| **SNR < 0 dB** | 信号强度低于噪声，通信质量较差 ⚠️ |

### 9.2 LoRa的独特优势

LoRa可以在**低于噪声底**的情况下解调信号：

| 扩频因子(SF) | 极限SNR | 说明 |
|-------------|---------|------|
| SF7 | -7.5 dB | 传输速率最快，距离最短 |
| SF8 | -10 dB | - |
| SF9 | -12.5 dB | - |
| SF10 | -15 dB | - |
| SF11 | -17.5 dB | - |
| SF12 | -20 dB | 传输速率最慢，距离最远 |

**关键点**：LoRa可以在SNR为-20dB时仍能解调信号，这是GFSK（需8dB）无法做到的。

### 9.3 信号质量判断标准

#### 快速判断表

| 等级 | RSSI | SNR | 说明 |
|------|------|-----|------|
| **优秀** ✅ | > -70 dBm | > 0 dB | 信号极强，通信稳定 |
| **良好** ✅ | -70 ~ -100 dBm | -7 ~ 0 dB | 信号正常，通信可靠 |
| **可用** ⚠️ | -100 ~ -115 dBm | -13 ~ -7 dB | 可通信，但余量小 |
| **边缘** ⚠️ | -115 ~ -120 dBm | -20 ~ -13 dB | 接近极限，可能丢包 |
| **不可用** ❌ | < -120 dBm | < -20 dB | 通信困难，需优化 |

#### 综合判断

**良好信号**：
- RSSI > -115 dB **且** SNR > -7 dB

**边缘信号**：
- RSSI ≤ -120 dB **或** SNR ≤ -13 dB

**特殊情况**：
| 现象 | 原因 | 建议 |
|------|------|------|
| RSSI好但SNR差 | 环境噪声高 | 检查干扰源，考虑更换频段 |
| RSSI差但SNR好 | 距离网关太远 | 增加网关或中继器 |

### 9.4 实际信号强度计算

**当 SNR > 0 dB 时**：
- RSSI = 实际信号场强

**当 SNR < 0 dB 时**：
- 实际信号强度 = RSSI + SNR

**原因**：当信号弱于噪声时，RSSI测得的是噪声的强度，而非信号本身。

### 9.5 环境噪声判断

| 噪声强度 | 等级 | 说明 |
|----------|------|------|
| > -90 dBm | 噪声偏高 ⚠️ | 可能存在干扰源 |
| -90 ~ -110 dBm | 噪声正常 ✅ | 正常环境 |
| < -110 dBm | 低噪声 ✅ | 理想环境 |

### 9.6 部署建议

1. **优先关注SNR**：比RSSI更能反映通信质量
2. **结合SF判断**：不同扩频因子对应不同的极限SNR
3. **多日测试**：SNR需连续监测多天确认稳定性
4. **考虑障碍物**：
   - 混凝土墙衰减约15-20 dB
   - 金属物对信号影响最大
5. **保持菲涅尔区清洁**：网关与水表之间尽量无遮挡

### 9.7 项目应用案例

**如何查看水表信号质量**：
1. 登录Comcore平台
2. 设备管理 → 水表设备档案
3. 找到目标水表，查看历史数据
4. 查看RSSI和SNR字段

**排查思路**：
```
水表数据未上报
    ↓
查看RSSI/SNR
    ↓
RSSI < -120 dBm？
    ↓ 是                    ↓ 否
距离太远，增加网关      SNR < -13 dB？
                            ↓ 是            ↓ 否
                        噪声干扰        通信正常
                        排查干扰源      检查其他原因
```

## 十、实际案例：哥伦比亚LoRaWAN车载测试

### 10.1 测试背景

**项目**：哥伦比亚超声波燃气表LoRaWAN通信测试

**测试需求**：
- 山区环境
- 高人口密度区域
- 两到三层地下空间

**测试方式**：车载移动测试

### 10.2 关键发现

#### LoRaWAN不适合车载移动通信 ❌

**结论**：
- LoRaWAN **不适合** 车载移动通信
- 需要**定点通信**

**原因分析**：
1. LoRaWAN采用Class A模式（水表常用），上行后开两个接收窗口
2. 移动状态下，信号强度变化快，无法稳定建立通信链路
3. 多普勒效应影响信号质量
4. 车速导致切换网关频繁，丢包率上升

#### 优化建议

**缩短上传报文**，提升测试成功率：
- 减少payload长度
- 使用更低的Data Rate（更高SF）
- 增加重传机制

## 十一、影响LoRaWAN通信的因素与优化方案

### 11.1 主要影响因素

#### 11.1.1 距离与路径损耗

| 因素 | 影响 | 说明 |
|------|------|------|
| **传输距离** | RSSI衰减 | 距离越远，信号越弱 |
| **菲涅尔区** | 信号遮挡 | 障碍物在菲涅尔区内会衰减信号 |

**典型距离衰减**（自由空间）：
- 距离增加1倍，信号衰减约6dB

#### 11.1.2 障碍物穿透损耗

| 材质 | 衰减值 | 说明 |
|------|--------|------|
| **玻璃** | 2-5 dB | 衰减最小，穿透性好 ✅ |
| **木材/木门** | 2-3 dB | 衰减小 |
| **砖墙** | 8-15 dB | 中等衰减 |
| **混凝土墙** | 20-35 dB | 衰减严重 ⚠️ |
| **钢筋混凝土** | 25-40 dB | 衰减严重，内含金属 ⚠️ |
| **金属** | 35+ dB | 几乎完全阻断 ❌ |

**实测数据**（来自本项目测试）：
| 场景 | 穿透能力 | RSSI/SNR范围 |
|------|----------|--------------|
| 地下负一层 | ✅ 可穿透 | -79~-109 dBm / 2.5~10.75 dB |
| 地下负二层 | ✅ 可穿透 | -110~-118 dBm / -13~3.75 dB |
| 地下负三层 | ❌ 无法穿透 | 超出能力 |

#### 11.1.3 干扰类型

| 干扰类型 | 原因 | 表现 |
|----------|------|------|
| **同频干扰** | 同一频段、同一SF同时传输 | 数据包碰撞，丢包率上升 |
| **邻频干扰** | 相邻信道信号泄漏 | 解调失败 |
| **外部干扰** | Wi-Fi、蓝牙、微波炉等设备 | SNR下降 |
| **多径干扰** | 信号反射、折射 | RSSI波动>10dB，信号失真 |

**环境噪声判断**：
| 噪声强度 | 等级 | 建议 |
|----------|------|------|
| > -90 dBm | 噪声偏高 ⚠️ | 排查干扰源 |
| -90 ~ -110 dBm | 正常 ✅ | 可正常部署 |
| < -110 dBm | 低噪声 ✅ | 理想环境 |

#### 11.1.4 移动状态

**结论**：LoRaWAN **不适合车载移动通信** ❌

**原因**：
1. Class A模式需要稳定的上下行时序
2. 移动中信号变化快，链路不稳定
3. 多普勒效应影响信号质量
4. 网关切换频繁，丢包率上升

**解决方案**：定点安装，或改用NB-IoT/4G

#### 11.1.5 频段选择

| 频段 | 穿透能力 | 适用场景 |
|------|----------|----------|
| **433/470 MHz** | 最强 ✅ | 亚洲，障碍物多的环境 |
| **868 MHz** | 中等 | 欧洲 |
| **915 MHz** | 较弱 | 北美、南美 |
| **2.4 GHz** | 最弱 | 全球通用，但干扰严重 |

**关键**：低频段穿透能力比高频段高约**45%**

#### 11.1.6 扩频因子(SF)

| SF | 速率 | 距离 | 抗干扰 | 穿透 | 空中时间 |
|----|------|------|--------|------|----------|
| SF7 | 最快 | 最近 | 弱 | 弱 | 最短 |
| SF10 | 中等 | 中等 | 中 | 中 | 中等 |
| SF12 | 最慢 | 最远 | 强 ✅ | 强 ✅ | 最长 |

**选择建议**：
- 近距离、高密度：SF7-SF9
- 远距离、障碍多：SF10-SF12

### 11.2 优化方案

#### 11.2.1 网关部署优化

| 优化项 | 方法 | 效果 |
|--------|------|------|
| **高度** | 网关置于高处（>5m） | 提升2-5dB |
| **位置** | 居中放置，避开遮挡 | 最大化覆盖 |
| **密度** | 每1-3km²部署1网关 | 减少盲区 |
| **冗余** | 部署多个网关协同 | 提升可靠性 |

**本项目实测**：
- 山区（障碍少）：单网关覆盖2.78km
- 城市高密度：单网关覆盖约2km
- 工业区：单网关覆盖约1km

#### 11.2.2 天线优化

| 天线类型 | 增益 | 适用场景 |
|----------|------|----------|
| 鞭状天线 | 2-3 dBi | 近距离，室内 |
| 玻璃钢天线 | 5-8 dBi | 中距离，户外 ✅ |
| 定向天线 | 10-15 dBi | 远距离，固定方向 |

**安装建议**：
1. 天线远离金属物体和厚墙
2. 靠近窗户利用玻璃低衰减特性
3. 高增益天线适合水平远距离通信

#### 11.2.3 参数优化

| 参数 | 优化方法 | 说明 |
|------|----------|------|
| **ADR** | 启用自适应数据速率 | 自动调整SF和功率 |
| **发射功率** | 根据距离调整 | 近端降低，远端提高 |
| **信道选择** | 避开干扰频段 | 现场频谱扫描 |
| **重传机制** | 启用确认机制 | 提升可靠性 |

#### 11.2.4 覆盖扩展方案

| 方案 | 适用场景 | 实现方式 |
|------|----------|----------|
| **中继器** | 信号死角 | 定点中继节点 |
| **Mesh组网** | 复杂环境 | 多跳路由 |
| **室内网关** | 地下室/封闭空间 | 独立部署 |

### 11.3 不同场景部署建议

#### 11.3.1 城市高密度区域

**挑战**：建筑遮挡、干扰多

**优化方案**：
- 网关密度：每1-2km²部署1台
- SF选择：SF9-SF11
- 天线：5-8dBi玻璃钢天线
- 高度：楼顶或高处

**本项目实测**：青岛市区，覆盖约2km，9个点位全成功

#### 11.3.2 工业区

**挑战**：金属设备多、多径干扰严重

**优化方案**：
- 网关密度：每0.5-1km²部署1台
- SF选择：SF10-SF12
- 天线：定向天线指向关键区域
- 中继：必要时部署中继节点

**预期覆盖**：50-200米

#### 11.3.3 地下空间

**挑战**：混凝土穿透损耗大

**优化方案**：
- 单层地下：可穿透，SF11-SF12
- 多层地下：部署室内网关或中继
- 负三层以上：建议改用其他方案

**本项目实测**：
- 负一层/负二层：可通信
- 负三层：无法通信

#### 11.3.4 山区/农村

**挑战**：地形遮挡、距离远

**优化方案**：
- 网关高度：山顶或高处
- SF选择：SF11-SF12
- 天线：高增益定向天线
- Mesh：多跳扩展覆盖

**本项目实测**：山区环境，单网关覆盖2.78km，11个点位全成功
